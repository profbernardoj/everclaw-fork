## Home Mini Setup Notes

New Mac Mini for agent work. Instructions from Thomas B. (BasedAI) for SSH security hardening:

### SSH Key Setup
1. Generate Ed25519 key pair: `ssh-keygen -t ed25519 -a 100`
2. Copy to server: `ssh-copy-id -i ~/.ssh/id_ed25519.pub username@remote_host`

### SSH Config (`/etc/ssh/sshd_config`)
- `PasswordAuthentication no` — force SSH keys only
- `PermitRootLogin no` — no direct root login
- `PubkeyAuthentication yes` — enable public keys
- `MaxAuthTries 3` — limit brute-force attempts
- `AllowUsers your_username` — restrict to specific accounts

### Optional: Change Default Port
- Change `#Port 22` to `Port <your_chosen_number>` (e.g., 2244)
- Reduces automated scan noise by ~99%

### macOS Firewall
1. System Settings > Network > Firewall → On
2. Options: "Block all incoming connections" → Off (SSH needs this)
3. Verify `sshd-keygen-wrapper` is allowed

### Fail2Ban (Optional)
- `brew install fail2ban` — bans IPs after failed login attempts
- Thomas skipped this initially

### Apply Changes
```bash
sudo launchctl unload /System/Library/LaunchDaemons/ssh.plist
sudo launchctl load -w /System/Library/LaunchDaemons/ssh.plist
# Or just restart
```
